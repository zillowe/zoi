Zoi provides a dedicated suite of commands for registry maintainers to initialize, manage, and audit package registries. These commands are grouped under the `zoi registry` (alias `reg`) subcommand.

## Core Commands

### `zoi registry init [path]`

Initializes a new, empty Zoi registry in the specified directory. It scaffolds the foundational directory structure and configuration files.

* **scaffolds:** `repo.yaml`, `packages.json`, `advisories.json`.
* **directories:** `core`, `main`, `community`, `test`, `archive`, `zillowe`.

```sh
zoi registry init ./my-new-registry
```

### `zoi registry add-package <name> --repo <tier>`

(alias: `add-pkg`)

Creates a new package scaffold in the specified repository tier.

* **Args:** `<name>` (package name), `--repo <tier>` (e.g. `community`, `main`).
* **Creates:** A directory and a minimal, commented `.pkg.lua` file.

```sh
zoi registry add-package my-tool --repo community
```

### `zoi registry add-advisory <package-name>`

(alias: `sec`)

Interactively prompts for vulnerability details and creates a temporary security advisory file.

* **Fields:** Summary, Severity, Affected Range, Fixed Version, Description, Reference URL.
* **Creates:** A `[PREFIX]-[YEAR]-TEMP.sec.yaml` file in the package directory.

```sh
zoi registry add-advisory my-tool
```

### `zoi registry generate-metadata`

(alias: `gen-meta`)

Scans the registry for all package definitions and security advisories to generate the central index files.

* **Packages:** Updates `packages.json` with metadata from all `.pkg.lua` files.
* **Advisories:** Assigns permanent IDs to `TEMP` advisory files and updates `advisories.json`.
* **ID Assignment:** Uses the `advisory_prefix` and current year from `repo.yaml`.

```sh
zoi registry generate-metadata
```

### `zoi registry check`

(alias: `lint`, `audit`)

Performs a comprehensive validation of the local registry.

* **Checks:** Every `.pkg.lua` file is linted for metadata consistency and valid dependency syntax.
* **Validation:** Ensures all referenced repositories exist and `repo.yaml` is valid.

```sh
zoi registry check
```

## Registry Structure

A standard Zoi registry follows this layout:

```text
my-registry/
├── repo.yaml           # Registry configuration and tiers
├── packages.json       # Generated package index (v1)
├── advisories.json     # Generated advisory index (v1)
├── core/               # Essential packages
│   └── hello/
│       └── hello.pkg.lua
├── community/          # User-submitted packages
│   └── my-tool/
│       ├── my-tool.pkg.lua
│       └── ZSA-2026-D0001.sec.yaml
└── ...
```

## Maintenance Workflow

1. **Initialize:** Create your registry with `zoi reg init`.
2. **Add Package:** Use `zoi reg add-pkg` to start a new package.
3. **Edit:** Modify the generated `.pkg.lua` to define your build and install logic.
4. **Audit:** Run `zoi reg check` to ensure everything is correct.
5. **Index:** Run `zoi reg gen-meta` to update the JSON indexes for clients.
6. **Commit:** Commit your changes to Git and push to your registry host.

For more details on package creation, see the [Creating Packages](/docs/zds/zoi/creating-packages) guide.
